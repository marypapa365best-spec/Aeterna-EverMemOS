## 今天主要做了什么

### 1. 大模型与设置

- **Gemini** 作为分身对话的 LLM，并支持 **OpenAI** 备选。
- 在页面「**设置**」里可以填写并保存：
  - EverMemOS API Key（记忆云端）
  - Gemini API Key、OpenAI API Key（分身对话）
- 显示名称放在设置第一项，两个 API 放在下面同一类。

### 2. 进化聊天室修好并能用

- 之前不能聊天、发送按钮灰掉 → 已去掉「必须先点唤醒」的限制，可以直接发消息。
- 报错「**回复失败：对话请求失败（404）」** → 做了两件事：
  - 后端：支持从请求头/body 读 API Key，并正确转发到 Gemini/OpenAI。
  - 前端：新增 **TwinChat** 组件，**浏览器直连 Gemini API**，不经过本机后端，避免 404。
- 模型报错「gemini-1.5-flash is not found」→ 已改为 **Gemini 2.5 Flash**（前端直连 + 后端代理都改）。
- 对话界面乱 → 改成**对话聊天模式**：左右气泡、头像、清晰的用户/助手区分。

### 3. 分身能「看到」EverMemOS 里的记忆

- 之前 Gemini 说「我没办法直接进入 evermindos 查找」→ 现在会在**每次发消息时**：
  - 用当前输入去 EverMemOS **拉取相关记忆**（搜索 + 最近列表兜底）；
  - 把记忆整理成一段文字，写进发给 Gemini 的 **systemInstruction**；
  - 分身回复时就能基于这些记忆说话。
- 记忆给大模型时：优先用 EverMemOS 的 **summary / episode / atomic_fact** 等简短字段，没有则截断 content（约 400 字），避免塞太长原文。

### 4. 分身「只能看到最近一条记忆」的问题

- 原因：存记忆时没有统一 **group_id**，或拉取时按 group 过滤太严，导致只看到一条。
- 处理：
  - **存记忆**：后端统一带 `group_id: twinId`、`group_name: "Personal Twin"`，保证同分身同组。
  - **拉记忆**：拉取列表时不再默认带 `group_id`，按 user 维度拿，这样历史记忆都能被搜到、注入对话。

### 5. 腾讯云部署方案（已写好，明早做）

- 目标：**打开网页就能用**，不用在评委/用户本机配后端、改 .env。
- 做法：
  - **方式一（推荐）**：服务器上 `npm run build` 得到 `dist/`，用 **一个 Node 进程**（`npm run start` 或 PM2）同时提供前端静态页和 `/api` 接口。
  - 可选方式二：Nginx 反代 + Node，适合要 80/443 或 HTTPS 的情况。
- API Key：不强制在服务器配 .env，用户打开网页后在「设置」里填即可。

### 6. 用 GitHub 方便以后更新

- 建议把代码推到 **GitHub**，服务器用 `git clone` 拉取。
- **以后每次改代码**：本地改完 → `git push` → 服务器上 `git pull` → `npm ci` → `npm run build` → `pm2 restart twin`，即可更新线上。
